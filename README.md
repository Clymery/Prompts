# Prompts

这是一个用于收藏和沉淀高质量提示词的个人仓库。

当前阶段只维护三个方向：

- `Image`: 图片生成提示词
- `skill`: 技能类提示词
- `vedio`: 视频生成提示词

后续如果内容规模变大，再考虑增加更细的分类、索引文件或自动化工具。

## 当前目录结构

```text
Prompts/
├── README.md
├── Image/
│   ├── 人物主体/
│   ├── 动漫风/
│   │   ├── 复古系列/
│   │   ├── 宫崎骏/
│   │   └── 新海诚/
│   ├── 海报/
│   └── 风景/
├── skill/
└── vedio/
```

## 目录说明

| 目录 | 用途 |
| --- | --- |
| `Image/人物主体/` | 存放人物、肖像、角色主体、模特写真等图片提示词案例 |
| `Image/动漫风/` | 存放动漫风格图片提示词案例，可按具体风格继续细分 |
| `Image/海报/` | 存放商业海报、活动海报、产品宣传图等图片提示词案例 |
| `Image/风景/` | 存放自然风景、城市景观、空间氛围等图片提示词案例 |
| `skill/` | 存放技能类提示词，例如写作、分析、总结、编程、学习、工作流等 |
| `vedio/` | 存放视频生成相关提示词，例如镜头运动、分镜、场景节奏、角色动作等 |

> `vedio` 当前按仓库已有目录保留。若后续想统一英文拼写，可以改为 `video`。

## 文件命名建议

建议使用简短、可检索的英文命名：

```text
主题-用途-版本.md
```

示例：

```text
fashion-poster-v1.md
product-photography-v1.md
writing-assistant-v1.md
cinematic-camera-move-v1.md
```

命名规则：

- 使用小写英文
- 单词之间用 `-`
- 重要迭代使用 `v1`、`v2`
- 一个文件只放一个主要 prompt 或一个完整案例

## 图片 Prompt 组织方式

图片类 prompt 建议采用“提示词 + 示例效果 + 参数记录”合并在同一个 Markdown 文件中的方式。

也就是说，不建议把同一个案例拆成 `prompt/` 和 `example/` 两个文件。一个完整案例就是一个文件，这样查看 prompt 时可以马上看到它的效果、参数和使用心得。

推荐结构：

```text
Image/
├── 人物主体/
│   └── editorial-portrait-v1.md
├── 动漫风/
│   ├── 宫崎骏/
│   │   └── warm-countryside-scene-v1.md
│   └── 新海诚/
│       └── rainy-city-sunset-v1.md
├── 海报/
│   └── product-launch-poster-v1.md
└── 风景/
    └── misty-mountain-morning-v1.md
```

单个图片案例文件可以使用下面的结构：

````md
# 标题

## 用途

说明这个 prompt 适合生成什么类型的图片。

## Prompt

```text
在这里放可直接复制使用的图片提示词。
```

## 参数

```text
模型、比例、风格参数、质量参数等。
```

## 负面提示词

```text
不希望出现的内容、画面问题、质量问题等。
```

## 示例效果

可以放生成结果说明，也可以引用图片路径。

```text
示例：画面主体清晰，背景层次丰富，整体偏电影感，但人物手部容易出错。
```

## 示例图片

```text
如果有图片，可以记录相对路径，例如：
./outputs/editorial-portrait-v1-01.png
```

## 备注

- 适合的场景
- 容易出错的地方
- 可替换关键词
````

如果某个 prompt 后续产生了很多输出图，再在同级目录下增加一个对应的资源目录即可，例如：

```text
Image/海报/product-launch-poster-v1.md
Image/海报/product-launch-poster-v1/
├── output-01.png
├── output-02.png
└── notes.md
```

## 技能 Prompt 记录格式

技能类提示词建议放在 `skill/`，可以用于沉淀某类可复用能力。

````md
# 标题

## 适用场景

说明这个 prompt 解决什么问题。

## Prompt

```text
在这里放提示词正文。
```

## 输入示例

```text
给模型的示例输入。
```

## 输出要求

```text
希望模型如何组织答案、格式、语气或步骤。
```

## 备注

- 可替换变量
- 使用心得
- 适用边界
````

适合收录的技能类 prompt 包括：

- 写作和改写
- 总结和提炼
- 编程辅助
- 方案分析
- 学习辅导
- 工作流拆解

## 视频 Prompt 记录格式

视频类提示词建议放在 `vedio/`，重点记录画面变化和镜头语言。

````md
# 标题

## 用途

说明这个 prompt 适合生成什么类型的视频。

## Prompt

```text
在这里放视频生成提示词。
```

## 画面设定

- 主体:
- 场景:
- 风格:
- 光线:
- 色彩:

## 镜头运动

```text
推近、拉远、环绕、跟拍、俯拍、慢动作等。
```

## 时长与比例

```text
例如 5s、10s、16:9、9:16、1:1。
```

## 备注

- 适合的平台或模型
- 容易失败的细节
- 可替换关键词
````

## 收录标准

一个 prompt 值得放进仓库，最好满足以下任意几项：

- 在真实任务中效果稳定
- 能明显提升输出质量
- 有清晰使用场景
- 可以复用或改写
- 有示例或参数记录
- 有失败经验或优化记录

暂时不建议收录：

- 只有一句泛泛指令的 prompt
- 没有场景说明的“咒语”
- 无法判断效果的片段
- 来源不清晰且可能涉及版权或隐私的内容

## 维护方式

1. 先判断 prompt 属于 `Image`、`skill` 还是 `vedio`。
2. 图片类 prompt 按主题放到 `Image/人物主体/`、`Image/动漫风/`、`Image/海报/` 或 `Image/风景/`。
3. 技能类 prompt 直接放到 `skill/`。
4. 视频类 prompt 直接放到 `vedio/`。
5. 每个文件尽量同时保留用途、正文、参数和示例效果。
6. 重要 prompt 迭代时用 `v1`、`v2` 区分版本。

## 后续可扩展方向

目前先保持轻量。等内容变多之后，可以再考虑：

- 增加 `index.md` 做总索引
- 增加标签系统
- 增加模型对比记录
- 增加失败案例和优化过程
- 将 `vedio` 统一更名为 `video`
